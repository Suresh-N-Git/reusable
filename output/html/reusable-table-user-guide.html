ReusableTable User Guide

Install and use the reusable Angular Material table library

What It Is

ReusableTable is an Angular library that provides a configurable Material table with search, sorting, pagination, column toggling, export, row actions, and an optional footer row for totals and counts.

What You Need

  • Angular 18 project
  • Node.js and npm
  • Material and Bootstrap styles, as used by the repo

Required Packages

  • @angular/material and @angular/cdk for the table UI
  • bootstrap for the styling included in the app
  • xlsx for Excel export
  • jspdf and jspdf-autotable for PDF export
  • file-saver for download flows used by the app

Install

1. Install the package and its dependencies.

npm install reusable-table-0.0.1.tgz

2. Import the library module.

import { ReusableTableModule } from 'reusable-table';

3. Add the shared styles from the library.

@import 'reusable-table/styles';

Configure Columns

Define a column array and choose the type for each field. The most important part is matching the column type to the shape of data your users will pass in. The guide and repo support: text, integer, number, currency, date, actions, chip, multiline, and link. 

{
  id: 'col6',
  name: 'Files',
  width: '20%',
  type: 'link',
  displayField: 'UploadedFileName',
  linkField: 'RelativeFilePath'
}

Supported Data Shapes

These examples show the values users should provide for each common column type. The raw object shape matters, especially for multiline rows and link rows. 

// Text / regular string
{
  id: 'name',
  name: 'Name',
  width: '20%'
}
// Data: { name: 'Pump A' }

// Integer
{
  id: 'id',
  name: 'Id',
  width: '5%',
  type: 'integer',
  align: 'right'
}
// Data: { id: 1 }

// Number with decimal places
{
  id: 'flow',
  name: 'Flow',
  width: '10%',
  type: 'number',
  align: 'right',
  digits: '1.2-2'
}
// Data: { flow: 1000.9999 }

// Multiline array
{
  id: 'col5',
  name: 'Notes',
  width: '20%',
  type: 'multiline',
  displayField: 'TextToDisplay'
}
// Data: {
//   col5: [
//     { TextToDisplay: 'Plain text' },
//     { TextToDisplay: 'Second line' }
//   ]
// }

// Link array
{
  id: 'col6',
  name: 'Files',
  width: '20%',
  type: 'link',
  displayField: 'UploadedFileName',
  linkField: 'RelativeFilePath'
}
// Data: {
//   col6: [
//     {
//       UploadedFileName: 'table-export.pdf',
//       RelativeFilePath: 'https://example.com/table-export.pdf'
//     }
//   ]
// }

// Actions
{
  id: 'actions',
  name: 'Actions',
  width: '5%',
  type: 'actions',
  align: 'center',
  actions: {
    select: { show: true, color: 'primary' },
    edit: { show: true, color: 'accent' },
    delete: { show: true, color: 'warn' }
  }
}

Prepare Data

Pass a normal array of objects into the table. For nested values, use displayField and linkField to control what appears and where it links. For decimals, use type: 'number' and digits such as '1.2-2'. 

const pumpData = [
  {
    id: 1,
    name: 'Pump A',
    flow: 1000.99,
    col5: [{ TextToDisplay: 'Plain text' }],
    col6: [{ UploadedFileName: 'table-export.pdf', RelativeFilePath: 'https://example.com/table-export.pdf' }]
  }
];

Use The Component

Bind your data, columns, table config, and row event handlers.

<app-reusabletable
  [data]="pumpData"
  [columns]="reUsableTableColumn"
  [tableConfig]="reUsableTableConfig"
  (rowEdit)="onEdit($event)"
  (rowDelete)="onDelete($event)"
  (rowSelect)="onSelect($event)">
</app-reusabletable>

Action Button Disable Rules

Each action button (select, edit, delete) supports an optional disableWhen rule. The rule names a key on the row and a value to compare against. When row[key] === equals, the button is rendered but disabled — clicks do not fire the corresponding row event. 

{
  id: 'rowOps',
  type: 'actions',
  name: 'Actions',
  width: '10%',
  actions: {
    edit: {
      show: true,
      color: 'accent',
      disableWhen: { key: 'approved', equals: true }
    },
    delete: { show: true, color: 'warn' },
    select: { show: true, color: 'primary' }
  }
}

The equals value is matched with strict equality, so the consumer declares the exact value their data uses:

  • equals: true — boolean flags
  • equals: 'Y' — Y/N flags
  • equals: 'Yes' — Yes/No flags
  • equals: 'locked' — status strings

Actions without a disableWhen rule are always enabled. Existing column configs without this field continue to work unchanged.

What The Host Component Must Declare

In the consuming component, users must declare the column definition array and the table config object. These are the two key inputs the table needs to know what to render and which features to enable. 

reUsableTableColumn: ReUsableTableColumn[] = [];
reUsableTableConfig: ReusableTableConfig = {
  searchTextMode: 'displayed',
  appearance: {
    zebraColor: '#bfd4f1',
    hoverColor: '#e3f2fd',
    selectedRowColor: '#ffe0b2',
    headingToPrint: 'New Heading'
  },
  pagination: {
    enabled: true,
    defaultPageSize: 5
  },
  sorting: {
    enabled: true
  },
  toolbar: {
    showSearch: true,
    showColumnToggle: true,
    showCsv: true,
    showExcel: true,
    showPdf: true,
    showPrint: true
  },
  footer: {
    enabled: true,
    sticky: true
  }
};

Footer Row

The table supports an optional footer row that can display aggregated values, counts, or static labels per column. Footer values react to the active filter — they reflect only the rows currently in view, so totals update automatically as users search.

Enable the footer in the table config:

footer: {
  enabled: true,
  sticky: true   // keep footer visible when the table scrolls
}

Declare per-column footer behavior via the footer field on the column definition:

// Sum of numeric values (formatted using the column's digits)
{ id: 'flow', name: 'Flow', type: 'number', digits: '1.2-2',
  footer: { type: 'sum' } }

// Count of currently visible rows
{ id: 'id', name: 'Id', type: 'integer',
  footer: { type: 'count' } }

// Fixed label
{ id: 'name', name: 'Name',
  footer: { type: 'text', value: 'Totals →' } }

// Custom aggregation
{ id: 'amount', name: 'Amount',
  footer: {
    type: 'custom',
    formatter: (rows) => `Max: ${Math.max(...rows.map(r => r.amount))}`
  } }

Available footer types:

  • sum, avg, min, max — numeric aggregations. Non-numeric values are ignored. If no numeric values are found, the cell renders empty and a warning is logged to the console in development.
  • count — number of currently visible rows.
  • text — renders the provided static string.
  • custom — receives the filtered rows array and returns a string.

Columns without a footer field render an empty footer cell, which keeps the footer row aligned with the header and body. Footer values are cached and recomputed only when data, filter, or visible columns change — they do not recalculate on every change detection cycle.

Run The Demo App

  • npm install
  • npm start
  • Open http://localhost:4200/
  • The root route redirects to /tableui in this repo.

Notes

  • The component supports search, sorting, pagination, column toggle, CSV/Excel/PDF export, print, an optional footer row, and per-row action button disable rules.
  • Regular text columns use the raw field value unless you add a special type or formatter.
  • For multiline and link columns, the table expects arrays of objects, not a single string.
  • Export behavior in the repo depends on xlsx, jspdf, and jspdf-autotable.
  • The demo app also uses Angular Material icons, form-field, input, menu, checkbox, button, tooltip, chips, sort, and paginator modules.
  • Not found in repo: a published npm registry URL, versioned release notes, or an end-user onboarding flow.
