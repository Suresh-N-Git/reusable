ReusableTable User Guide

Configurable Angular Material data table — search, sort, paginate, export, footer aggregations, and per-row action disable rules.

What It Is

ReusableTable is an Angular library wrapping Angular Material’s MatTable with a declarative column and config API. A consumer defines columns and a config object, passes a data array, and gets a table with search, sorting, pagination, column toggle, CSV / Excel / PDF export, print, an optional footer row, and per-row action button disable rules.

The library is built on Angular 18 and Material 18. The component is OnPush; expensive per-row work (footer aggregations, search filtering) is cached and recomputed only when inputs change.

Requirements

  • Angular 18 project
  • Node.js and npm
  • @angular/material and @angular/cdk
  • bootstrap for the styling used by the demo app
  • xlsx, jspdf, jspdf-autotable, file-saver for exports

Install

Install from a packed tarball:

npm uninstall reusable-table
npm install reusable-table-0.0.1.tgz

Import the module in the consuming app:

import { ReusableTableModule } from 'reusable-table';

Optional shared styles:

@import 'reusable-table/styles';

Quick Start

Three things the host component must declare: columns, config, data.

// component .ts
reUsableTableColumn: ReUsableTableColumn[] = [
  { id: 'id',   name: 'Id',   type: 'integer', width: '10%', align: 'right' },
  { id: 'name', name: 'Name', width: '40%' },
  { id: 'flow', name: 'Flow', type: 'number', digits: '1.2-2', align: 'right', width: '20%' },
  {
    id: 'rowOps', type: 'actions', name: 'Actions', width: '15%',
    actions: {
      edit:   { show: true, color: 'accent' },
      delete: { show: true, color: 'warn' }
    }
  }
];

reUsableTableConfig: ReusableTableConfig = {
  searchTextMode: 'displayed',
  pagination: { enabled: true, defaultPageSize: 25 },
  sorting: { enabled: true },
  toolbar: { showSearch: true, showColumnToggle: true,
             showCsv: true, showExcel: true, showPdf: true, showPrint: true }
};

pumpData = [
  { id: 1, name: 'Pump A', flow: 1000.99 },
  { id: 2, name: 'Pump B', flow: 250.50  }
];
<!-- component .html -->
<app-reusabletable
  [data]="pumpData"
  [columns]="reUsableTableColumn"
  [tableConfig]="reUsableTableConfig"
  (rowEdit)="onEdit($event)"
  (rowDelete)="onDelete($event)"
  (rowSelect)="onSelect($event)">
</app-reusabletable>

Column Definition

Every column shares a base shape. Type-specific fields are listed in the next section.

FieldTypePurpose
idstring Data key on the row object. Required and must be unique within the column array.
namestring Header label shown in the table. Required.
typesee types card Render contract. Defaults to plain text.
widthstring CSS width hint, e.g. '20%' or '120px'.
align'left' | 'center' | 'right' Cell content alignment. Header and footer align with the cell.
digitsstring Decimal format for type: 'number', e.g. '1.2-2'.
styleRecord Inline style applied to every cell in this column.
chipStyleRecord Background and text styling for type: 'chip'.
displayFieldstring Nested key shown to the user for multiline and link.
linkFieldstring Nested key holding the URL for type: 'link'.
exportFormatter(value) => string Override export-time rendering for this column.
actionssee Actions card Required for type: 'actions'.
footersee Footer card Per-column footer cell behavior.

Column Types

The type field controls how a cell renders. Default is plain text.

// 'text'  — default. Renders the field value as a string.
{ id: 'name', name: 'Name', width: '20%' }

// 'integer' — formatted whole number.
{ id: 'id', name: 'Id', type: 'integer', align: 'right', width: '5%' }

// 'number' — formatted decimal. Use 'digits' to control places.
{ id: 'flow', name: 'Flow', type: 'number', digits: '1.2-2', align: 'right' }

// 'currency' — formatted as INR currency.
{ id: 'price', name: 'Price', type: 'currency', align: 'right' }

// 'date' — formatted as dd-MM-yyyy.
{ id: 'createdAt', name: 'Created', type: 'date' }

// 'chip' — value rendered inside a styled badge.
{ id: 'status', name: 'Status', type: 'chip',
  chipStyle: { backgroundColor: '#3f51b5', color: 'white' } }

// 'multiline' — array of objects, one line per item.
{ id: 'col5', name: 'Notes', type: 'multiline', displayField: 'TextToDisplay' }
// data: { col5: [{ TextToDisplay: 'first line' }, { TextToDisplay: 'second line' }] }

// 'link' — array of objects, each rendered as a hyperlink.
{ id: 'col6', name: 'Files', type: 'link',
  displayField: 'UploadedFileName', linkField: 'RelativeFilePath' }
// data: { col6: [{ UploadedFileName: 'a.pdf', RelativeFilePath: 'https://...' }] }

// 'actions' — see Actions card. Renders icon buttons per row.
{ id: 'rowOps', type: 'actions', name: 'Actions',
  actions: { edit: { show: true, color: 'accent' } } }

The actions column is identified by type: 'actions', not by id. The id can be anything you want — 'rowOps', 'controls', etc.

Table Configuration

Pass a ReusableTableConfig object as [tableConfig]. All sections are optional; defaults are applied per missing field.

reUsableTableConfig: ReusableTableConfig = {

  searchTextMode: 'displayed',  // or 'all' — see note below

  appearance: {
    zebraColor:        '#bfd4f1',
    hoverColor:        '#e3f2fd',
    selectedRowColor:  '#ffe0b2',
    headingToPrint:    'Pump Inventory'
  },

  pagination: {
    enabled: true,
    defaultPageSize: 25         // 5 | 10 | 25 | 100
  },

  sorting: {
    enabled: true
  },

  toolbar: {
    showSearch:       true,
    showColumnToggle: true,
    showCsv:          true,
    showExcel:        true,
    showPdf:          true,
    showPrint:        true
  },

  footer: {
    enabled: true,
    sticky:  true               // keep footer pinned during scroll
  },

  selection: {
    multiSelect: true           // default false — see below
  }

};

searchTextMode: 'displayed' matches only the rendered text (what the user sees). 'all' also searches hidden fields like linkField for link columns.

selection — row selection and highlighting new

Controls how clicked rows are highlighted. Driven entirely by the select action button; edit and delete clicks never modify the highlighted set.

  • multiSelect: false (default) — single-row highlight. Clicking any action button (select, edit, delete) highlights the clicked row, replacing whatever was previously highlighted.
  • multiSelect: true — multi-row highlight. Each click on the select button toggles that row in or out of the highlighted set. Edit and delete clicks fire their events but do not modify the highlighted set.

Two-way binding — the parent owns the array

For multi-select usage, the parent component owns the selected-rows array. The library exposes [selectedRows] as an input and (selectedRowsChange) as an output, so two-way binding works out of the box: 

<app-reusabletable
  [data]="pumpData"
  [columns]="reUsableTableColumn"
  [tableConfig]="reUsableTableConfig"
  [(selectedRows)]="selectedItems"
  (rowSelect)="onSelect($event)"
  (rowEdit)="onEdit($event)"
  (rowDelete)="onDelete($event)">
</app-reusabletable>

With the binding in place, every parent-side mutation of selectedItems automatically updates the highlights — no method calls into the library, no @ViewChild, no synchronisation logic: 

selectedItems: any[] = [];

removeFromList(item: any): void {
  this.selectedItems = this.selectedItems.filter(i => i !== item);
}

clearAll(): void {
  this.selectedItems = [];
  this.pumpData = [...this.pumpData];   // optional data refresh
}

uploadAll(): void {
  this.api.uploadMany(this.selectedItems);
}

The select-button click in the table emits a new array via selectedRowsChange; the parent stores it; the library reads it back via [selectedRows]. Single source of truth, no drift.

Single-select consumers (multiSelect: false) do not need to bind [(selectedRows)] at all — the library handles single-row highlighting internally.

Persistence and tracking

Selection persists across pagination, sorting, and search filtering — rows are tracked by object reference, not by index. A row that scrolls out of view due to a filter remains in the highlighted set and reappears highlighted when the filter is cleared.

The highlight color comes from appearance.selectedRowColor. Set it explicitly if the default does not contrast enough with your zebra striping.

Actions Column

Three action types are supported: select, edit, delete. Each is independently shown / hidden, colored, and (optionally) disabled per row by a rule. 

{
  id: 'rowOps',
  type: 'actions',
  name: 'Actions',
  width: '10%',
  actions: {
    select: {
      show: true,
      color: 'primary'
    },
    edit: {
      show: true,
      color: 'accent',
      disableWhen: { key: 'approved', equals: true }
    },
    delete: {
      show: true,
      color: 'warn',
      disableWhen: { key: 'status', equals: 'final' }
    }
  }
}

disableWhen — per-row disable rule new

Each action accepts an optional disableWhen rule that names a key on the row and a value to compare against. When row[key] === equals, the button is rendered but disabled — clicks do not fire the corresponding row event.

The equals value is matched with strict equality:

  • equals: true — boolean flags
  • equals: 'Y' — Y/N flags
  • equals: 'Yes' — Yes/No flags
  • equals: 'locked' — status strings

Actions without a disableWhen rule are always enabled. Existing column configs without this field continue to work unchanged.

Footer Row

Optional footer row showing aggregations, counts, or static labels per column. Footer values react to the active filter — they reflect only the rows currently in view, so totals update automatically as users search.

Enable in the table config:

footer: { enabled: true, sticky: true }

Declare per-column behavior via the footer field:

// Sum of numeric values, formatted with the column's digits
{ id: 'flow', type: 'number', digits: '1.2-2',
  footer: { type: 'sum' } }

// Count of currently visible rows
{ id: 'id', type: 'integer',
  footer: { type: 'count' } }

// Fixed label
{ id: 'name',
  footer: { type: 'text', value: 'Totals →' } }

// Custom aggregation
{ id: 'amount',
  footer: {
    type: 'custom',
    formatter: (rows) => `Max: ${Math.max(...rows.map(r => r.amount))}`
  } }

Available footer types:

  • sum, avg, min, max — numeric aggregations. Non-numeric values are ignored. If no numeric values are found, the cell renders empty and a warning is logged to the console in development.
  • count — number of currently visible rows.
  • text — renders the provided static string.
  • custom — receives the filtered rows array and returns a string.

Columns without a footer field render an empty footer cell. Footer values are cached and recomputed only when data, filter, or visible columns change — not on every change detection cycle.

Print and Export Behavior

Print and exports always include the full filtered set, regardless of the active page size.

  • No filter, any page size: full dataset is exported / printed.
  • Filter active: only matching rows are exported / printed.
  • Filter matches zero rows: a “No Data Found.” alert appears and no file is produced.
  • Footer enabled: the footer row is included in CSV, Excel, PDF, and print outputs.
  • Print: opens a new window with a printable rendering of the table; the window self-closes once the print dialog is dismissed.
  • Action column: excluded from CSV / Excel / PDF (kept on screen and in print).

Events

The component emits four outputs:

  • (rowSelect) — emits the row object when its select button is clicked.
  • (rowEdit) — emits the row object when its edit button is clicked.
  • (rowDelete) — emits the row object when its delete button is clicked.
  • (selectedRowsChange) — emits the new selected-rows array whenever the select button toggles membership. Pair with [selectedRows] for two-way binding.

A button disabled via disableWhen does not fire its event.

Defaults

Sensible defaults are applied for any config field the consumer omits.

  • searchTextMode: 'all'
  • pagination.defaultPageSize: 25
  • sorting.enabled: true
  • toolbar.*: all features visible
  • footer.enabled: false
  • selection.multiSelect: false
  • appearance.zebraColor: '#f5f5f5'

Run The Demo App

  2. npm start

Notes

  • The component supports search, sorting, pagination, column toggle, CSV / Excel / PDF export, print, an optional footer row, per-row action button disable rules, and single- or multi-row selection highlighting.
  • Regular text columns use the raw field value unless you add a special type or formatter.
  • For multiline and link columns, the table expects arrays of objects, not single strings.
  • The actions column is identified by type: 'actions'; its id is free.
  • Print and exports operate on the full filtered set — pagination does not limit them.
  • Search highlighting is applied via safe HTML escaping. User input is escaped before regex use, so special characters in the search box match literally.
